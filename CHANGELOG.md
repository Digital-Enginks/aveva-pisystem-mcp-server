## [1.0.0]

Initial stable release.

### Added

- Stateless, zero-trust **Model Context Protocol (MCP) server** exposing the AVEVA PI System Web API to MCP clients.
- **Outbound authentication modes** to PI Web API: `anonymous`, `basic`, `kerberos` (Negotiate/SPNEGO), and `bearer` (OIDC via AVEVA Identity Manager).
- **Read tools** over time-series archives, AF asset databases, elements, attributes, and summaries.
- **Write tools**, globally **default-deny** (`MCP_READ_ONLY=true` and `MCP_WRITE_TOOLS_ENABLED=false`) with a per-call scope/role gate.
- **Inbound transports:** `stdio` (default, local trust boundary) and `http` with required edge authentication (bearer JWT via JWKS, or mTLS).

### Security

- **Zero-trust posture:** outbound TLS verification enforced in production, over HTTP transport, and with pinning (`NODE_TLS_REJECT_UNAUTHORIZED=0` always blocked at startup; `PIWEBAPI_TLS_VERIFY=false` allowed only as a non-production debugging escape hatch); private-CA trust via `PIWEBAPI_TLS_CA_FILE` unioned with public roots.
- **Single fixed service identity** to PI Web API; inbound caller identity is not propagated upstream.
- **Two-axis auth error-channel:** inbound failures surface as protocol-level errors; all tool-execution failures (including upstream PI 401/403) surface as sanitized `isError` tool results with stable AppError codes.
- Allowlist-based error sanitizer, structured redacted audit logging, and inbound rate limiting / outbound concurrency bounds.
